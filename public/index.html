meme-api

静态站点（Cloudflare Pages）。素材在仓库 原图/ 内编号生成 *.webp / *.gif；npm run build 将 原图/ 镜像到 public/meme/ 用于部署。直链可被外部引用；JSON 已配置 CORS。

浏览图库（分页、按文件夹分类、点击复制链接）

获取表情包列表

GET /memes.json（与旧版兼容：GET /api/v1/memes，内部映射为同一份 JSON）

返回所有分类及文件相对路径；前端与脚本可用其拼接直链。

curl -sS ""

直链访问单张表情包

部署后文件在 public/meme/<分类>/（与仓库 原图/<分类>/ 内容一致），对应 URL：

GET /meme/<分类>/<文件名>.webp 或 .gif

https://example.com/meme/%E7%99%BD%E8%89%B2%E5%B0%8F%E4%BA%BA/1.webp

中文或多段路径请使用编码后的 URL（浏览器会自动处理，图库「复制链接」亦为已编码地址）。

维护流程

  1. 在 原图/<分类>/ 放入未编号素材；运行 python convert_to_webp.py（就地生成编号文件，成功后会删除这些未编号源图；若需保留源文件可加 --keep-sources）。
  2. 若仍有「动图 WebP」编号文件，可在 原图/ 下运行 python animated_webp_to_gif.py 转为 .gif。
  3. npm run build 将 原图/ 镜像到 public/meme/ 并生成 public/memes.json，再部署 public/。
